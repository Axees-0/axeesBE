### **Interactive Front-End Demonstration with Simulated User Cursor**

Your goal is to create an **interactive front-end demo** that showcases the desktop application created in **PROMPT_01_UI_DESIGN.txt**, along with the guided tour functionality from **PROMPT_02_GUIDED_TOUR.txt**. The demo must include a **simulated cursor** that moves naturally, interacting with UI elements in real time as if a real user were operating it, while demonstrating both the application's features and the guided tour functionality.

**Output File Naming Convention:**
Your deliverable should use this naming convention for consistency with previous components and be placed in a "/demo" folder:
- Main demo HTML file: `/demo/PROJECT_DEMO_MAIN.html`
- Demo CSS styles: `/demo/PROJECT_DEMO_STYLES.css`
- Demo JavaScript file: `/demo/PROJECT_DEMO_SCRIPT.js`
- Python HTTP server script: `/demo/PROJECT_DEMO_SERVER.py`
- Any additional assets: `/demo/PROJECT_DEMO_[PURPOSE].[ext]`

The demo should integrate with and demonstrate both the UI files from PROMPT_01_UI_DESIGN.txt and the tour functionality from PROMPT_02_GUIDED_TOUR.txt.

---

### **1. Simulated Cursor with Realistic User Behavior**
The cursor should behave like a human user by:
- **Moving smoothly** across the screen with easing effects.
- **Clicking buttons** naturally, with slight variations in timing.
- **Filling out forms** character by character, imitating human typing speed.
- **Toggling settings** (switches, checkboxes, radio buttons) based on scripted interactions.
- **Navigating menus**, scrolling, and hovering over tooltips before selecting options.

The cursor should **pause briefly** before interacting with elements, mimicking real user hesitation.

---

### **2. Fully Functional UI Interactions**
The demo must feature a **dynamic UI** that reacts as the actual app would:
- **Buttons trigger actions** (e.g., opening modals, submitting forms, triggering animations).
- **Input fields validate text** in real time and show errors where applicable.
- **Toggles and checkboxes change state** with smooth animations.
- **Dropdown menus expand and collapse** naturally when hovered over or clicked.
- **Mouse-over effects** should provide hover states and tooltips where relevant.

Use **mock data** to create realistic form submissions and API responses.

---

### **3. Seamless, Self-Contained Setup**
The demo should be:
- **Visually similar to the final product** while remaining lightweight.
- **Minimal dependency-based** (prefer vanilla JavaScript, CSS animations, or lightweight libraries).
- **Responsive** to different screen sizes and devices.

Include a **Python HTTP server script** for easy local testing.

---

### **4. Hands-Free Execution and Graceful Termination**
To provide a hands-free experience, the demo should:
- **Launch automatically** upon execution.
- **Ensure a seamless start-up** with minimal manual setup on macOS/Linux (e.g., by executing a simple startup script or command).
- **Automatically terminate and clean up** any temporary states once the demo sequence is completed, instead of creating any persistent file.
- Optionally, display a **"Demo Complete"** message or transition to a summary screen before closing the interface.

---

### **Execution Flow:**
1. **Launch the demo** – the interface loads automatically.
2. A **simulated cursor moves across the UI, interacting with elements in real time**.
3. The cursor performs a series of scripted interactions, mimicking a user exploring the app.
4. Each UI element responds **exactly as it would in production** (e.g., animations, validations, UI feedback).
5. The demo **runs hands-free**, providing an engaging and realistic user experience.
6. At the end of the demo, the interface **gracefully terminates** or resets itself without generating any persistent files.

---

### **Key Requirements for the Simulated Cursor**
- **Smooth movement with easing effects** (no robotic jumps).
- **Randomized delays between actions** to appear more natural.
- **Realistic typing simulation** with variable speeds.
- **Hover interactions before clicks** where applicable.
- **Scroll interactions** to mimic user navigation on long pages.

This demonstration should feel **authentic, immersive, and intuitive**, providing a **realistic preview** of how users will interact with the app without leaving behind any extraneous files once complete.
