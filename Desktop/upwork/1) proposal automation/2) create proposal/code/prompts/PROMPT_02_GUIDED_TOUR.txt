### **Industry-Style Guided Tour for a Web-Based Application (Single File Implementation with Auto-Start)**  

Create a **self-contained guided tour** that highlights key UI elements step by step in a **single file** for easy integration with the UI design from **PROMPT_01_UI_DESIGN.txt**. The tour should start **automatically when the page loads**, using **visual cues such as red outlines around elements, instructional tooltips, and smooth transitions** to guide the user through the desktop application.  

**Output File Naming Convention:**
Your deliverable should use this naming convention for consistency with the UI design files and be placed in a "/demo" folder:
- Main tour JavaScript file: `/demo/PROJECT_TOUR_SCRIPT.js`
- Any CSS needed specifically for the tour: `/demo/PROJECT_TOUR_STYLES.css`
- If using a standalone demo HTML: `/demo/PROJECT_TOUR_DEMO.html`

The tour must be designed to easily integrate with the UI files from PROMPT_01_UI_DESIGN.txt.

---

### **1. Guided Tour with Auto-Start & Countdown**  
- The tour should **automatically begin** as soon as the page loads.  
- Before highlighting each UI element, display a **5-second countdown timer** inside the tooltip.  
- The tour should **auto-advance** to the next step when the countdown reaches **0**, ensuring a hands-free experience.  

---

### **2. Single File Implementation & UI Highlights**  
- The entire guided tour, including **HTML, CSS, and JavaScript**, should be in **one file** for easy embedding.  
- **Red Outlines & Focus Effects:** Each highlighted element should have a **red outline**, subtle animations (**pulsing or glowing effect**) to draw attention.  
- **Instructional Tooltips:**  
  - Tooltips should appear **next to each highlighted element**, explaining its function and purpose.  
  - Each tooltip should display a **step progress indicator** (e.g., "Step 3 of 7").  

---

### **3. Hands-Free, Seamless Navigation**  
- The tour should **require no manual interaction**—each step auto-progresses.  
- **Smooth transitions** ensure an engaging experience.  
- The **tour follows a logical user journey**, introducing features in a structured sequence.  
- **Dynamic Adjustments:**  
  - The tooltip position should adjust dynamically to **avoid overlap** with elements.  
  - The tour should adapt to **different screen sizes** for responsiveness.  

---

### **4. Execution & Compatibility**  
- The tour must run **without external dependencies**, using only vanilla HTML, CSS, and JavaScript.  
- The **single-file** solution should be easy to embed in any web application.  
- The tour should **gracefully exit** at the last step or provide an option to **restart**.  

---

### **Final Deliverable:**  
- A **fully functional, single-file guided tour** that starts **automatically when the page loads**.  
- A **professional, engaging, and intuitive** walkthrough experience.  

This ensures an **effortless, interactive onboarding** experience without requiring user input.
