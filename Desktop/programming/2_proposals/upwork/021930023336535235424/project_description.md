Summary
     My organization currently has a handful of data "pipelines" that rely on a lot of repetitive analyst labor that I am looking to automate. I was doing this myself (a competent data scientist and product owner but not an expert data engineer) but other issues require my attention, so I need to farm this out.

Our current process (15 years old) involves our members sending us spreadsheets via email, which we upload to on-prem Access dbs. There is some picky business logic but everything is pretty straightforward from an engineering standpoint.

I want to have AWS handle all of this. Automated retrieval of the email attachments, store them in S3, clean/transform with glue, then dump into a data lake or db. Finally, I'll need to send out periodic reports (mimicking our current reports, I have examples) via email and host/make-available some basic Power BI dashboards. Our data is not particularly large or complex. Our availability/latency requirements are nothing special either. If you're suggesting something innovative, you're almost certainly over-thinking this. We do not need to migrate our historical data at this time, we can just use fresh data as it comes in.

Here are the tough parts:
1. I need to make considerable, demonstrable progress on this within 4 weeks. I need you have plenty of bandwidth available to focus on this for this short duration.
2. I need to manage this closely. Probably daily stand ups. Very tight decision loops. I will also be contributing to the code base. 
3. You need to take the appropriate amount of initiative and push back on my direction where appropriate. We need to build rapport quickly. Don't block yourself. Ask questions, make smart decisions with imperfect information. Don't always build what I ask for, build what I need. 

You don't need to be too concerned with the visualizations/Power BI deliverables. Provided we end up with a well-structured DB that I can query, I can do that part later.
